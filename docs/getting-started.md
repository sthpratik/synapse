# Getting Started

This guide will help you get up and running with Synapse in minutes.

## Prerequisites

- Node.js 16 or higher
- K6 installed ([Installation Guide](k6-installation.md))

## Installation

Install Synapse globally:

```bash
npm install -g synapse
```

Verify installation:

```bash
synapse --help
```

## Quick Start

### Simple Load Test (No Configuration)

For immediate testing without creating configuration files:

```bash
synapse test --url "https://jsonplaceholder.typicode.com/posts" --concurrent 10 --requests 100
```

This command will:
- Generate a basic K6 script
- Run 100 requests with 10 concurrent users
- Display results immediately

### Advanced Configuration

### 1. Initialize a Project

Create a new load test configuration:

```bash
synapse init --name "My First Test" --url "https://jsonplaceholder.typicode.com/posts"
```

This creates a `synapse.yml` file in your current directory.

### 2. Review Configuration

The generated configuration looks like this:

```yaml
name: "My First Test"
description: "Generated by Synapse CLI"
baseUrl: "https://jsonplaceholder.typicode.com/posts"

execution:
  mode: "construct"
  concurrent: 10
  iterations: 100

parameters:
  - name: "id"
    type: "integer"
    min: 1
    max: 1000

k6Options:
  thresholds:
    http_req_duration: ["p(95)<500"]

request:
  method: "GET"
  headers:
    "User-Agent": "Synapse Load Tester"
```

### 3. Run Your First Test

Execute the load test:

```bash
synapse run
```

This will:
- Generate a K6 JavaScript file
- Execute the load test
- Display results
- Save metrics to `./output/results.json`

## Understanding the Results

Synapse will show K6 output including:
- Request rate and duration
- Success/failure rates
- Threshold violations
- Performance metrics

## Common Commands

```bash
# Quick test without configuration
synapse test --url "https://api.example.com" --concurrent 5 --requests 50

# Validate configuration
synapse validate

# Generate K6 script without running
synapse generate --output test.js

# Run with custom output directory
synapse run --output ./my-results

# Dry run (generate script only)
synapse run --dry-run --keep-script
```
